# קוראים אחרת v2

גרסת v2 של `Read for Real` היא כלי קריאה אקדמית עם ארבעה רכיבים משלימים:

1. סימון משפטים לפי תפקיד טיעוני: טענה, ראיה, הסתייגות.
2. בדיקה עצמית בסוף כל חלק עם הסברים על סימונים נכונים, שגויים ומשפטים שפספסתם.
3. כתיבה סוקרטית קצרה בין חלקי המאמר.
4. מבחן קצר בסיום, לפני הדוח המסכם.

העיקרון הפדגוגי פשוט: לא רק "להבין את הטקסט", אלא ללמוד איך טיעון אקדמי עובד מבפנים. לכן הכלי מאט בכוונה את הקריאה, דורש סימון ידני, כולל בדיקה עצמית לפני ההמשך, מוסיף מבחן קצר בסוף, וחוסם הדבקה בשאלות הכתיבה.

זהו פיילוט שמיועד גם למרצות ומרצים אחרים: אפשר להחליף את התוכן של המאמר, מפתח הסימון, השאלות והמבחן בעיקר דרך `_article_data.js`, בלי לעבור לפרויקט build-heavy.

## מה עובד כרגע

- קריאת המאמר המלא באנגלית, מחולק לחלקים, פסקאות ומשפטים
- תרגום עברי לכל פסקה בלחיצה
- סימון משפטים דרך תפריט בחירה קטן
  - לחיצה על משפט פותחת בחירה מפורשת
  - טענה / ראיה / הסתייגות / הסר
- בדיקה עצמית בסוף כל חלק
  - נפתחת אחרי שסומנו לפחות הקטגוריות שמופיעות במפתח של אותו חלק
  - נועלת את הסימונים בחלק
  - מציגה משוב inline על סימונים נכונים, שגויים, מיותרים ומשפטים שפספסתם
- שאלות סוקרטיות בין חלקי המאמר
  - חסימת paste/copy/cut/drop
  - מינימום 50 תווים
- מבחן קצר בסוף
- דוח מסכם עם:
  - ציון סימון כולל ולפי קטגוריה
  - תשובות סוקרטיות
  - ציון מבחן
  - זמן קריאה ופתיחת תרגומים
- מצב review מהדוח
  - "חזרו למאמר עם הסברים"
  - כל משפט שמופיע במפתח מסומן
  - לחיצה על משפט פותחת הסבר של המרצה

## מבנה הקבצים בפועל

```text
index.html                  ← נקודת הכניסה של v2
app.js                      ← הלוגיקה של האפליקציה
styles.css                  ← כל הסגנונות
_article_data.js            ← המאמר, מפתח הסימון, שאלות ומבחן
v2-screen-copy.md           ← נוסחי המסכים המעודכנים
read-for-real-v2-spec (1).md← הספק המקורי
read-for-real-v2.jsx        ← טיוטת React מוקדמת, לא ה-entry הפעיל
```

## הערה על מפתח הסימון

האפליקציה קוראת כרגע את ההסבר לכל משפט מהשדה הבא, לפי סדר עדיפות:

1. `fb`
2. `feedbackMissed`
3. `feedbackCorrect`

כלומר, אם בעתיד רוצים לאחד את המבנה ל-`fb` בלבד, הקוד כבר מוכן לזה.

## איך הזרימה נראית

1. ברוכים הבאים
2. הוראות
3. חלק קריאה
4. בדקו תשובות
5. שאלת כתיבה
6. חלק הבא
7. מבחן קצר
8. דוח מסכם
9. חזרו למאמר עם הסברים

## מה חשוב לעבור עליו ידנית

- עברית בתרגומים
  - יש עדיין מקומות שדורשים ליטוש אנושי, במיוחד במונחים פילוסופיים
- מפתח הסימון
  - לוודא שלא סומנו יותר מדי משפטים כטענות או ראיות
- הסברים (`fb`)
  - ההסבר צריך לומר למה המשפט חשוב, לא רק מה הקטגוריה שלו
- שאלות סוקרטיות
  - כדאי לוודא שהן באמת מחייבות חשיבה, לא רק שליפה
- מבחן
  - כדאי לוודא שהשאלות בודקות הבנה מושגית ולא רק זיכרון של פרט בודד
- דוח להורדה
  - לוודא שייצוא ה-HTML נשאר RTL ושהאיים האנגליים נשארים קריאים

## הערת מימוש

למרות שקיימת בתיקייה טיוטת React (`read-for-real-v2.jsx`), הגרסה הפעילה כרגע היא אפליקציה סטטית פשוטה של `HTML + JS + CSS`. זה מכוון: קל יותר לערוך, לבדוק, ולפרוס אותה כמו שהיא.
